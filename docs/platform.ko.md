# 플랫폼 지원

[← README](../README.ko.md) · [English](platform.md) | **한국어**

반응 코어(상태·애니메이션·배회·드래그/던지기·트레이)는 이식 가능해요. 창 통합(올라타기/가림·
클릭-포커스)은 세 타깃 플랫폼 모두에서 동작해요 — KDE는 KWin 스크립트, Windows는 폴링
기반 Win32 API, macOS는 폴링 기반 Quartz(`pyobjc`). KDE 전용으로 남은 건 작업표시줄
숨김뿐이에요. 그 외 미구현 기능은 그냥 꺼져요 — 펫은 여전히 실행돼요.

세 플랫폼 모두 v1.0.0 기준 **실기 검증**을 마쳤어요.

| 플랫폼 | 실행 | 창 통합 |
|--------|------|---------|
| **KDE Plasma** (Wayland/X11) | ✅ | ✅ 전부 — 올라타기, 가림 클리핑/숨김, 클릭-포커스, 작업표시줄 숨김 |
| 기타 Linux (GNOME 등) | ✅ (XWayland) | ✖ KDE 전용 부분 no-op (배회/드래그/상태/트레이는 동작) |
| **Windows** | ✅ | ✅ 올라타기, 가림 클리핑/숨김, 클릭-포커스(`SetForegroundWindow`, 폴링 기반 `ctypes`/Win32); ✖ 작업표시줄 숨김 미구현 |
| **macOS** | ✅ | ✅ 올라타기, 가림 클리핑/숨김, 클릭-포커스(`osascript`); CG→Qt 좌표 런타임 자동 보정(폴링 기반 Quartz, `pyobjc`); ✖ 작업표시줄 숨김·호스트 최소화 시 숨김 미구현 |

CLI 도구(`bin/*`)는 전부 **Python** — bash 없음 — 이라 Python 되는 곳이면 어디서든 실행돼요.
KDE·Windows·macOS 모두 실사용 지원 대상이에요. GNOME은 창 통합 범위 밖이고요. 창 통합이
미구현인 곳(GNOME, 또는 macOS에서 pyobjc가 없는 경우)에선 바탕화면 바닥으로 폴백하고 해당
기능은 꺼져요.

> **macOS 테스트에 관하여.** 유지관리자에게 Mac이 없어서 macOS는 협력자가 실기로 검증해요 —
> 즉 macOS 전용 회귀는 릴리스 *전*보다 *후*에 드러나는 경향이 있어요. 리포트 정말 환영해요
> (아래 [macOS 참고](#macos-참고)).

## 요구사항

- Python 3 + PyQt6 — `pip install PyQt6`
- 전체 기능엔 **KDE Plasma**: `qdbus6`(창 통합/클릭-포커스), `wmctrl`(선택, 작업표시줄에서
  펫 숨김). Wayland면 XWayland.
- **Windows**: 추가 요구사항 없음 — 창 통합은 `src/claudlet/windows_win32.py`에서 표준
  라이브러리 `ctypes`로 `user32`/`dwmapi`/`kernel32`를 직접 호출해요.
- **macOS 전용**: 올라타기/가림에 `pip install pyobjc-framework-Quartz`
  (`src/claudlet/windows_macos.py`). 인스톨러(`pipx install claudlet` 또는 한 줄 소스
  설치)가 자동으로 깔아줘요. 선택 사항 — 없어도 펫은 실행되고, 창 통합만 꺼져요.
  Windows/Linux에선 절대 필요 없고 import되지도 않아요.

## 알려진 한계 / 미해결 항목

- **Windows 디스플레이 배율** — Win32 지오메트리 피드는 각 창을 해당 모니터의 DPI
  (`GetDpiForWindow`)로 물리 픽셀 → Qt 논리 좌표로 변환해요. 그래서 단일 모니터의 125%/150%
  배율, 그리고 원점에서 정렬된 동일-배율 멀티모니터에선 올라타기/가림이 맞아요. **혼합-DPI
  멀티모니터**(화면마다 배율이 다름)는 각 화면의 논리 원점 보정이 실기기에서 필요 — 그전까진
  경계 넘어가며 올라타기가 어긋날 수 있어요.
- **멀티유저 호스트** — 펫의 명령 채널은 **인증 없는** 루프백 TCP 리스너예요. 공용 머신에선
  다른 로컬 사용자가 `127.0.0.1:<port>`에 붙어 펫을 조작/종료할 수 있어요 (상태·모션만 — 파일/
  코드 접근은 불가). 단일 사용자 데스크톱엔 무해하고, 필요하면 토큰 핸드셰이크가 계획된 수정이에요.

## macOS 참고

- **화면 기록(Screen Recording) 권한 (선택).** macOS 10.15부터 창 열거 API는 권한이 없으면
  다른 앱 창의 *제목*을 숨겨요(시스템 설정 → 개인정보 보호 및 보안 → 화면 기록에서 펫을
  실행하는 터미널/Python에 부여). 권한이 **없어도** 올라타기/가림은 동작해요 — 앱 이름으로
  폴백하거든요 — 그러니 제목 수준 정확도가 필요할 때만 주면 돼요.
- **올라타기 좌표가 어긋나면?** `claudlet-macos-diag`로 펫이 보는 macOS 창 좌표 원본을
  출력해 보세요. 높이가 틀리거나 보조 모니터에서 어긋나는 올라타기를 진단하는 가장 빠른
  방법이에요. 출력과 함께 이슈로 남겨주세요.
- **macOS 미구현:** 작업표시줄 숨김(설계상 KDE 전용), 세션별 호스트 창 추적(호스트 최소화 시
  숨김). 그 외 — 배회, 드래그/던지기, 상태, 트레이, 올라타기, 가림, 클릭-포커스 — 는 지원돼요.
